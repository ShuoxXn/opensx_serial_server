# 工作模式示例

本目录保存可复制的工作模式示例代码。示例文件只包含模式 `.c/.h` 模板，不包含模式注册、CMake 构建入口或 Web 前端入口。复制到 `main/modes/` 并按说明补齐这些改动后才会生效。

## 文件

| 文件 | 说明 |
| --- | --- |
| `mode_protocol_report_example.h` | 示例模式头文件 |
| `mode_protocol_report_example.c` | UART 数据进入模式队列后，同时演示 TCP、MQTT、HTTP、UDP 发送 |

## 使用方式

1. 复制示例文件到 `main/modes/`。
2. 按业务重命名文件、函数名、NVS key 和模式名。
3. 在 `main/include/sx_work_mode_ids.h` 添加模式名常量。
4. 在 `main/modes/sx_builtin_modes.c` 注册模式。
5. 在 `main/CMakeLists.txt` 的 `SRCS` 中加入新的模式 `.c` 文件。
6. 修改 `main/static/web.html`、`main/static/web.js` 和 `main/static/i18n/*.json`，让用户可以选择和配置该模式。
7. 执行 `idf.py build`。

完整步骤见 `docs/工作模式开发指南.md` 的“新增工作模式”和“Web 接入细节”。

注册示例：

```c
#include "mode_protocol_report_example.h"

static const sx_work_mode_config_ops_t protocol_report_config = {
    .load = mode_protocol_report_example_load_config,
    .save = mode_protocol_report_example_save_config,
};

static const sx_work_mode_t protocol_report_mode = {
    .name = WORK_MODE_PROTOCOL_REPORT,
    .label = "协议上报示例",
    .start = mode_protocol_report_example_start,
    .stop = mode_protocol_report_example_stop,
    .after_start = NULL,
    .on_uart_data = mode_protocol_report_example_on_uart_data,
    .config = &protocol_report_config,
};

ESP_ERROR_CHECK(sx_work_mode_register(&protocol_report_mode));
```

模式名示例，添加到 `main/include/sx_work_mode_ids.h`：

```c
#define WORK_MODE_PROTOCOL_REPORT "protocol_report"
```

示例代码用于说明框架调用方式，不代表产品默认业务逻辑。
