# API 说明

本文档描述当前固件框架保留的主要 HTTP API 和配置字段。

框架层仍保留 BSD socket/TCP 网络能力、UART/GPIO 驱动、Modbus 辅助能力、MQTT/HTTP 等协议模块。当前内置工作模式用于测试 TCP 透传链路、MQTT/TCP 协议回显链路和 HTTP 主动上报链路。

当前内置工作模式：

| 工作模式 | 说明 |
| --- | --- |
| `tcp_transparent` | 纯 TCP 透传测试模式 |
| `protocol_test` | 协议测试模式，支持 MQTT/TCP 回显，并每 1 秒通过 HTTP 协议配置 POST `test` |

旧的 `mqtt_tcp`、`modbus_tcp`、`modbus_rtu` 工作模式已经从工作模式注册表和构建入口中移除。

## 通用约定

- 请求和响应默认使用 JSON。
- 配置保存到 NVS namespace：`nvs_namespace`。
- 工作模式字段保存为：`w_mode`。
- 当前默认工作模式为：`tcp_transparent`。

## 工作模式侧协议 API

工作模式代码位于 `main/modes/`。模式侧调用 TCP、MQTT、HTTP、UDP 时，推荐统一使用：

```c
#include "sx_protocol_api.h"
```

主要接口：

```c
esp_err_t sx_protocol_tcp_send(sx_protocol_tcp_target_t target,
                               const uint8_t *data, size_t len);
esp_err_t sx_protocol_tcp_server_send(const uint8_t *data, size_t len);
esp_err_t sx_protocol_tcp_client_send(const uint8_t *data, size_t len);
esp_err_t sx_protocol_mqtt_publish(const char *topic, const uint8_t *data,
                                   size_t len, int qos, bool retain);
esp_err_t sx_protocol_http_post_configured(const uint8_t *data, size_t len,
                                           const char *content_type);
esp_err_t sx_protocol_udp_send_to(const char *host, uint16_t port,
                                  const uint8_t *data, size_t len);

/* 兼容旧模式的别名，新模式优先使用上面的接口。 */
esp_err_t sx_protocol_send_tcp_server(const uint8_t *data, size_t len);
esp_err_t sx_protocol_send_tcp_client(const uint8_t *data, size_t len);
```

TCP 发送按全局 `tcpconn` 选择当前 TCP Server/Client 通道，MQTT 发布依赖当前 MQTT Client 连接状态，HTTP POST 会读取全局 `use_http` 和 `http_url`。UDP 主动发送需要显式传入目标地址和端口。详细说明见：

```text
docs/工作模式开发指南.md
```

## 工作模式配置

### `POST /work_mode_set`

保存当前工作模式和通用工作参数。

最小请求：

```json
{
  "work_mode": "tcp_transparent",
  "mode_config": {}
}
```

`mode_config` 是工作模式私有配置对象，由当前模式自己的配置回调保存。当前内置 `tcp_transparent` 没有私有配置。

兼容历史字段：

```json
{
  "work_mode": "tcp_transparent",
  "mode_config": {},
  "poll_time": "30000"
}
```

响应：

```json
{
  "code": 200,
  "msg": "success"
}
```

字段说明：

| 字段 | 说明 | 当前取值 |
| --- | --- | --- |
| `work_mode` | 工作模式 | `tcp_transparent` |
| `mode_config` | 当前工作模式私有配置 | 当前 TCP 透传任务为空对象 |
| `poll_time` | 历史通用轮询间隔字段 | 毫秒，当前 TCP 透传任务不使用 |

如果 `work_mode` 不是已注册模式，后端返回 `400 invalid work_mode`。

### `GET /work_mode_info`

读取当前工作模式配置。

典型响应：

```json
{
  "work_mode": "tcp_transparent",
  "available_modes": [
    {
      "name": "tcp_transparent",
      "label": "TCP透传"
    },
    {
      "name": "protocol_test",
      "label": "协议测试"
    }
  ],
  "mode_config": {
    "transport": "tcp",
    "has_private_config": false
  },
  "poll_time": "30000"
}
```

说明：`poll_time` 是兼容历史字段，当前 TCP 透传模式不会使用它。`protocol_test` 的 `mode_config` 会返回 `mqtt_echo_topic`、`tcp_echo`、`http_test_payload`、`http_test_interval_ms` 等描述字段。

## TCP 配置

### `GET /tcp_info`

读取 TCP 协议配置。

典型响应：

```json
{
  "use_tcp": "1",
  "tcpconn": "0",
  "tcp_server": "192.168.1.100",
  "tcp_port": "8888",
  "tcp_client_count": 0,
  "tcp_server_conn": 0
}
```

字段说明：

| 字段 | 说明 | 取值 |
| --- | --- | --- |
| `use_tcp` | 是否启用 TCP | `0`/`1` |
| `tcpconn` | TCP 模式 | `0`=TCP Server，`1`=TCP Client |
| `tcp_server` | TCP Client 连接目标地址 | IP/域名 |
| `tcp_port` | TCP 端口 | 默认 `8888` |
| `tcp_client_count` | TCP Server 当前客户端数量 | 数字 |
| `tcp_server_conn` | TCP Client 是否已连接远端 | `0`/`1` |
| `reg_packet` | TCP Client 注册包内容 | 字符串 |
| `reg_format` | 注册包格式 | `hex`/字符串 |
| `heart_packet` | TCP Client 心跳包内容 | 字符串，默认设备 MAC |
| `heart_format` | 心跳包格式 | `hex`/字符串 |
| `heart_interval` | 心跳间隔 | 秒，默认 `30` |

当前内置 TCP 透传任务只使用 `0`/`1`。Modbus 能力保留在框架模块中，新的业务任务如需 Modbus，应在自己的 `modes/` 实现中显式调用 Modbus API，而不是复用旧的内置 Modbus 工作模式。

## Web/API 配置接口

Web 后端接口集中在 `main/web/sx_web_server.c` 注册。新增工作模式的 Web 接入见 `docs/工作模式开发指南.md`，配置字段归属见 `docs/配置字段说明.md`。

本节 URI 清单按 `main/web/sx_web_server.c` 中 `register_web_uris()` 当前注册项核对。

### 设备与网络信息

#### `GET /devinfo`

读取设备名称、MAC、IP、网络模式等基础信息。

#### `GET /runtime_status`

读取运行时状态，例如 AP 状态、外网状态、内存和运行时间等。

#### `GET /net_set_info`

读取网络配置。

#### `POST /net_set`

保存网络配置。

### `GET /mqtt_info`

MQTT 配置接口在代码中仍保留为协议能力，但当前内置工作模式 `tcp_transparent` 不会调用 MQTT。

MQTT 下发配置载荷如果包含 `mode_config`，后端会按当前或载荷中的 `work_mode` 调用 `sx_work_mode_save_config()`。也就是说，新增工作模式的私有配置仍然应该通过工作模式配置回调保存，不要在 MQTT 协议层直接写业务 NVS key。

实际返回字段包括：`use_mqtt`、`mqtt_type`、`mqtt_server`、`mqtt_port`、`mqtt_username`、`mqtt_password`、`mqtt_clientid`、`mqtt_sub_topic`、`mqtt_pub_topic`、`qos`、`retain`、`mqtt_send`、`mqtt_time`、`mqttconn`。

### `POST /mqtt`

保存 MQTT 通用协议配置。该接口走通用 NVS 表单保存流程，成功返回：

```json
{
  "msg": "success",
  "code": 200
}
```

### `GET /http_info`

HTTP 配置接口在代码中仍保留为协议能力。当前内置工作模式 `tcp_transparent` 不会调用 HTTP 上报；`protocol_test` 会在 `use_http=1` 且 `http_url` 已配置时，每 1 秒向该 URL POST 文本 `test`。

实际返回字段包括：`use_http`、`http_port`、`httpconn`、`http_url`、`http_header`、`http_time`。

### `POST /http`

保存 HTTP 通用协议配置。`protocol_test` 只使用其中的 `use_http` 和 `http_url`，发送方式固定为 HTTP Client POST。

## UART WebSocket

WebSocket 用于 Web 页面与设备进行 UART 数据交互、日志和调试显示。

当前 TCP 透传模式的数据主链路是：

```text
UART -> mode_tcp_transparent -> TCP Server/Client
TCP Server/Client -> UART
```

## 新增工作模式时的 API 约定

新增工作模式通常需要同步：

1. 在 `main/include/sx_work_mode_ids.h` 增加模式名。
2. 在 `main/modes/` 新增模式 `.c/.h` 文件。
3. 在 `main/modes/sx_builtin_modes.c` 注册模式。
4. 在 `main/CMakeLists.txt` 的 `SRCS` 中加入新的模式 `.c` 文件。
5. 在 `main/static/web.html` 增加可选项或配置区域。
6. 在 `main/static/web.js` 增加提交和加载逻辑。
7. 在 `main/static/i18n/*.json` 增加中英文文案。
8. 必要时在 `main/web/sx_web_server.c` 增加独立 API 或特殊字段校验。

工作模式 API 只应该保存配置和选择模式，不应该直接实现业务逻辑。具体业务逻辑应放在 `main/modes/`。
