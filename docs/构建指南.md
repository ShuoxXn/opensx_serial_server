# 构建指南

本文档说明如何构建当前 OpenShuoxin 串口固件框架。

本仓库现在只维护一个型号，不再维护多型号配置，也不再使用多项目同步方式。构建产物由本地 `idf.py build` 生成，不提交到源码仓库。

## 环境要求

- ESP-IDF 开发环境
- Python 和 ESP-IDF 依赖已正确安装
- 当前 shell 已加载 ESP-IDF 环境变量

如果 ESP-IDF 尚未加载，通常需要执行类似命令：

```bash
. $IDF_PATH/export.sh
```

具体路径以本机 ESP-IDF 安装位置为准。

## 项目结构

```text
shuoxin_serial_server/
  CMakeLists.txt
  sdkconfig
  partitions.csv
  components/
  main/
  docs/
  search/
```

核心固件代码在 `main/`：

| 目录 | 说明 |
| --- | --- |
| `main/app/` | 启动入口 |
| `main/drivers/` | UART/RS485、GPIO、Modbus 等驱动 |
| `main/network/` | 以太网、Wi-Fi、AP、DNS、网络状态 |
| `main/protocols/` | MQTT、TCP、UDP、HTTP Client |
| `main/web/` | Web 服务、REST API、WebSocket、OTA、日志 |
| `main/system/` | 工具函数、日志、定时任务、工作模式注册表 |
| `main/modes/` | 具体工作模式 |
| `main/static/` | Web 前端静态资源 |

## 构建

在项目根目录执行：

```bash
idf.py build
```

构建成功后，产物会生成在本地 `build/` 目录中。`build/` 是临时构建目录，不应提交到仓库。

常见产物包括：

```text
build/bootloader/bootloader.bin
build/partition_table/partition-table.bin
build/ota_data_initial.bin
build/<project>.bin
build/<project>.elf
build/<project>.map
```

## 清理构建产物

如需重新完整构建：

```bash
idf.py fullclean
idf.py build
```

也可以直接删除本地 `build/` 目录后重新构建。

## 配置

当前项目使用根目录 `sdkconfig` 作为默认配置。

不再维护以下历史文件：

```text
sdkconfig.bak
sdkconfig.old
configs/sdkconfig.sp501w
configs/sdkconfig.sp401w
configs/sdkconfig.sp301w
```

如果后续需要修改配置，推荐使用：

```bash
idf.py menuconfig
```

修改后确认 `sdkconfig` 中的变更，再提交必要配置。

## 分区表

分区表文件：

```text
partitions.csv
```

如果调整 OTA、NVS 或应用分区大小，需要同步检查：

- `partitions.csv`
- `sdkconfig`
- OTA 相关代码和 Web 升级逻辑

## Web 静态资源

Web 页面资源位于：

```text
main/static/
```

这些文件通过 `main/CMakeLists.txt` 的 `EMBED_FILES` 嵌入固件。修改 Web 页面后，重新执行：

```bash
idf.py build
```

即可把新资源打进固件。

## 工作模式开发

如果要新增业务工作模式，参考：

```text
docs/工作模式开发指南.md
```

工作模式 Web 接入和配置项说明参考：

```text
docs/工作模式开发指南.md
docs/配置字段说明.md
```

工作模式代码应放在：

```text
main/modes/
```

新增源文件后，需要加入 `main/CMakeLists.txt` 的 `SRCS`。

## 不提交的内容

以下内容是本地生成或历史备份，不应提交：

```text
build/
dist/
bin/
sdkconfig.bak
sdkconfig.old
size_files.json
*.elf
*.map
*.pyc
__pycache__/
```
