# NVS 配置项梳理

本文档按框架职责梳理当前主要 NVS 配置字段，便于后续清理 Web 表单和新增工作模式。

默认 namespace：

```text
nvs_namespace
```

工作模式字段：

```text
w_mode
```

## 框架公共配置

这些字段属于固件框架本身，和具体工作模式无关。

| 字段 | 说明 | 默认值 |
| --- | --- | --- |
| `host_names` | 设备显示名称 | `串口服务器` / Web 缺省时为 `以太网串口服务器` |
| `device_type` | 设备类型标识 | `SP501` |
| `device_sn` | 设备序列号 | `0000000000000` |
| `lgname` | Web 登录用户名 | `admin` |
| `lgpwd` | Web 登录密码 | `12345678` |
| `ntp_server` | NTP 服务器 | 空 |
| `ap_name` | AP 名称 | 空 |
| `ap_password` | AP 密码 | 空 |
| `ap_wait_time` | AP 开放等待时间 | `0` |
| `log_enabled` | 日志开关 | 代码中按需初始化 |

## 网络配置

这些字段由网络管理和 Web 网络页面使用。

| 字段 | 说明 | 默认值 |
| --- | --- | --- |
| `netconn` | 网络接口选择，历史值中 `2` 表示 Wi-Fi，否则默认 Ethernet | 空 |
| `is_dhcp` | IP 获取方式，`2` 表示静态 IP，其他值按 DHCP 处理 | `2` |
| `static_ip` | 静态 IP | `192.168.1.84` |
| `static_netmask` | 静态子网掩码 | `255.255.255.0` |
| `static_gateway` | 静态网关 | `192.168.1.1` |
| `static_dns1` | 首选 DNS | `8.8.8.8` |
| `static_dns2` | 备用 DNS | `114.114.114.114` |
| `wifi_ssid` | Wi-Fi SSID | 空 |
| `wifi_password` | Wi-Fi 密码 | 空 |

外网状态不保存到 NVS。当前运行时判断逻辑是 ping `www.baidu.com` 成功即认为外网可用。

## 工作模式配置

| 字段 | 说明 | 默认值 |
| --- | --- | --- |
| `w_mode` | 当前工作模式 | `tcp_transparent` |
| `poll_time` | 通用轮询间隔字段 | `30000` |

Web/API 和 MQTT 配置下发使用 `mode_config` 传递模式私有配置。`mode_config` 本身不是固定 NVS key，具体保存哪些 key 由当前工作模式的 `config->save/load` 回调决定。

`poll_time` 是历史通用字段。当前 `tcp_transparent` 不使用它，保留是为了降低 Web/API 兼容风险。

模式私有配置示例见：

```text
docs/工作模式开发指南.md
docs/modes/
```

## TCP 透传测试配置

当前内置工作模式 `tcp_transparent` 使用 TCP 协议配置。

| 字段 | 说明 | 默认值 |
| --- | --- | --- |
| `use_tcp` | 是否启用 TCP | `1` |
| `tcpconn` | TCP 模式，`0`=TCP Server，`1`=TCP Client | `0` |
| `tcp_server` | TCP Client 连接目标地址 | `192.168.1.100` |
| `tcp_port` | TCP 端口 | `8888` |
| `tcp_send` | 历史发送模式字段 | `0` |
| `tcp_time` | 历史定时发送间隔字段 | `5` |
| `reg_format` | TCP Client 注册包格式 | 由 Web 保存 |
| `reg_packet` | TCP Client 注册包内容 | 由 Web 保存 |
| `heart_format` | 心跳包格式 | 由 Web 保存 |
| `heart_packet` | 心跳包内容 | 由 Web 保存 |
| `heart_interval` | 心跳间隔 | 由 Web 保存 |

当前 TCP 透传测试模式只应该使用纯 TCP Server/Client。旧的 `tcpconn=2/3` Modbus TCP 变体不再作为内置应用任务入口。

## MQTT 协议能力配置

MQTT 作为框架协议能力保留，默认关闭。当前 `tcp_transparent` 不主动调用 MQTT。

| 字段 | 说明 | 默认值 |
| --- | --- | --- |
| `use_mqtt` | 是否启用 MQTT | `0` |
| `mqtt_type` | MQTT 类型字段，当前保留历史值 | `0` |
| `mqtt_server` | MQTT Broker 地址 | `mqtt.shuoxin-iot.com` |
| `mqtt_port` | MQTT 端口 | `1883` |
| `mqtt_username` | MQTT 用户名 | 空 |
| `mqtt_password` | MQTT 密码 | 空 |
| `mqtt_clientid` | MQTT Client ID | 设备 MAC 派生 |
| `mqtt_sub_topic` | 订阅主题 | 设备 MAC 派生 |
| `mqtt_pub_topic` | 发布主题 | 设备 MAC 派生 |
| `qos` | MQTT QoS | `0` |
| `retain` | MQTT retain | `0` |
| `mqtt_send` | 历史定时上报开关 | `0` |
| `mqtt_time` | 历史定时上报间隔 | `5` |

## HTTP 协议能力配置

HTTP 作为框架协议能力保留，默认关闭。当前 `tcp_transparent` 不主动调用 HTTP 上报。

| 字段 | 说明 | 默认值 |
| --- | --- | --- |
| `use_http` | 是否启用 HTTP | `0` |
| `httpconn` | 请求方式字段，当前 Web 只提供 POST | `1` |
| `http_url` | HTTP 目标 URL | 空 |
| `http_port` | 历史端口字段 | `5000` |
| `http_header` | HTTP Header 字段 | 空 |
| `http_time` | 历史定时上报间隔 | `5` |

串口配置由 UART 驱动使用。

| 字段 | 说明 |
| --- | --- |
| `baud_rate` | 波特率 |
| `data_bit` | 数据位 |

## 历史遗留配置

以下字段来自旧业务模式或旧设备信息接口。后续应结合调用关系继续确认是否删除、隐藏或迁移到自定义模式。

| 字段 | 当前建议 |
| --- | --- |
| `modbus_tcp_timeout` | 旧 Web 工作模式字段，已从当前内置 Web 主流程移除。Modbus 自定义任务需要时应通过自己的 `mode_config` 定义 |
| `m_count`、`m*_...` | 旧 Modbus 模板配置。框架保留 Modbus 能力，但默认 Web 主流程不再强调 |
| `thsensor` | 旧设备信息字段，待确认调用方 |
| `psensor` | 旧设备信息字段，待确认调用方 |
| `adapter_id` | 旧设备信息字段，待确认调用方 |
| `interface` | 旧设备信息字段，待确认调用方 |
| `power` | 旧设备信息字段，待确认调用方 |
| `dns1`、`dns2` | 旧设备信息字段；当前网络配置使用 `static_dns1/static_dns2` |

清理原则：

- 已删除工作模式专用字段，不应继续出现在默认 Web 主流程。
- 框架能力字段可以保留，但默认关闭。
- 新工作模式需要自己的字段时，应在文档中注明字段归属，避免继续把业务字段写成全局公共字段。
